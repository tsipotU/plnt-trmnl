## #1a — Catalog foundation (this PR's scope)

**In scope**
- New file `packages/api/catalog/plants.json` with ~30 curated entries (3-5 per category × 8 categories) — enough to unblock #2/#3/#4/#37/#39 for real testing
- Catalog schema TypeScript types at `packages/api/src/catalog/types.ts`
- Loader at `packages/api/src/catalog/loader.ts` — reads JSON on startup, builds in-memory search index
- `GET /api/catalog/search?q=...&limit=20` endpoint at `packages/api/src/routes/catalog.ts`
  - Searches `latin_name`, `aliases`, `common_names.en[]`, `common_names.nl[]`
  - Case-insensitive substring + token prefix match
  - Returns `{ slug, latin_name, category, primary_common_name }` per hit
- Tests for loader + search scoring + API endpoint

**Out of scope (deferred to #1b)**
- Filling to 250+ entries
- LLM-assisted generation script (lives in `scripts/catalog/generate.ts`, Emiel runs offline)
- Dutch-name audit
- Illustration slug pipeline (#54)

**Schema (draft, to be frozen in the PR):**

```ts
type CatalogEntry = {
  slug: string;                  // kebab-case, stable, matches illustration filename
  latin_name: string;
  aliases?: string[];            // previous Latin names, common misspellings
  common_names: { en: string[]; nl: string[] };
  category: "foliage" | "flowering" | "succulent" | "cactus" | "tree" | "fern" | "palm" | "air-plant";
  care: {
    base_interval_days: number;
    light_preference: "low" | "medium" | "bright_indirect" | "direct";
    placement_hints: string;
    toxicity: string;            // short sentence
    size_category: "small" | "medium" | "large" | "tree";
    difficulty: "beginner" | "intermediate" | "expert";
  };
  origin: string;
  common_conditions: string[];   // short names; detailed structure added in #3
  // fields added in follow-up issues:
  // light_profile  — #3
  // placement_tips — #3
  // conditions     — #3 (replaces common_conditions)
  // facts          — #4
  // lore, etymology — #37
};
```

Why freeze `common_conditions` as a placeholder now: later issues extend the schema, but #1a must not block waiting for them. #3 replaces the field with a structured array; loader version-stamps so migrations stay traceable.

**30 seed entries distribution:**
- foliage (5): Monstera deliciosa, Pothos (Epipremnum aureum), ZZ plant, Philodendron hederaceum, Peperomia obtusifolia
- flowering (4): Phalaenopsis orchid, Peace lily (Spathiphyllum), Anthurium andraeanum, African violet
- succulent (4): Aloe vera, Haworthia fasciata, Echeveria elegans, Jade plant (Crassula ovata)
- cactus (3): Christmas cactus (Schlumbergera), Mammillaria, Golden barrel (Echinocactus)
- tree (5): Fiddle leaf fig (Ficus lyrata), Rubber tree (Ficus elastica), Money tree (Pachira), Dracaena marginata, Bird of paradise (Strelitzia)
- fern (3): Boston fern, Maidenhair (Adiantum), Bird's nest (Asplenium)
- palm (3): Parlor palm (Chamaedorea), Kentia (Howea forsteriana), Areca (Dypsis lutescens)
- air-plant (3): Tillandsia ionantha, Tillandsia usneoides, Tillandsia xerographica

Each entry authored from memory + one quick cross-check, marked with `source: "hand-seeded"` in a top-level metadata object so #1b can distinguish hand vs generated.
