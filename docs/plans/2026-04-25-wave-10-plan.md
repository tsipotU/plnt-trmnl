```tsx
import { useState } from 'react';

export function PotSizeTooltip() {
  const [isOpen, setIsOpen] = useState(false);

  const guides = [
    { label: 'Small (~8–14 cm)', body: 'A spread hand from fingertip to thumbtip, edge-to-edge across the pot rim.' },
    { label: 'Medium (~14–20 cm)', body: 'About one-and-a-half spread hands across.' },
    { label: 'Large (~20–25 cm)', body: 'Two spread hands across.' },
    { label: 'Extra large (~25 cm+)', body: 'Wider than two spread hands.' },
  ];

  return (
    <div style={{ position: 'relative', display: 'inline-block' }}>
      <button
        type="button"
        onClick={() => setIsOpen(!isOpen)}
        aria-label="Pot size help"
        title="Pot size guide"
        style={{
          background: 'none', border: 'none', color: 'var(--text-secondary)',
          fontSize: 16, padding: '0 8px', cursor: 'pointer',
          display: 'inline-flex', alignItems: 'center', justifyContent: 'center',
        }}
      >
        ⓘ
      </button>
      {isOpen && (
        <div style={{
          position: 'absolute', top: '100%', left: 0, marginTop: 8,
          background: 'var(--bg-card)', border: '1px solid var(--border)',
          borderRadius: 8, padding: 16, zIndex: 1000, minWidth: 280,
          boxShadow: '0 4px 12px rgba(0, 0, 0, 0.3)',
        }}>
          <button type="button" onClick={() => setIsOpen(false)} aria-label="Close" style={{
            position: 'absolute', top: 8, right: 8, background: 'none', border: 'none',
            color: 'var(--text-secondary)', fontSize: 20, cursor: 'pointer', padding: '4px 8px',
          }}>×</button>
          <div style={{ paddingRight: 24 }}>
            <p style={{ fontSize: 13, color: 'var(--text-secondary)', marginBottom: 12, fontStyle: 'italic' }}>
              Measure across the top of the pot, edge to edge.
            </p>
            {guides.map((g, i) => (
              <div key={i} style={{ marginBottom: i < guides.length - 1 ? 12 : 0 }}>
                <p style={{ fontSize: 13, fontWeight: 600, color: 'var(--text-primary)', marginBottom: 4 }}>
                  {g.label}
                </p>
                <p style={{ fontSize: 12, color: 'var(--text-secondary)' }}>{g.body}</p>
              </div>
            ))}
          </div>
        </div>
      )}
    </div>
  );
}
```

## Task 3: Post-add confirmation rethink (#130)

**Files:**
- Modify: `packages/api/client/src/components/EnrichmentSplash.tsx` — repurpose 'enriching' mode + new 'fallback' mode
- Modify: `packages/api/client/src/pages/AddPlant.tsx` — drive the new state machine

**Why:** Today the splash shows an indefinite "✨ Enriching {name}..." spinner that never resolves under pull-based enrichment if no AI tool is connected. Post-Wave 8, ~most adds resolve via the catalog within milliseconds. Repurpose the splash:

- **Catalog match** (most cases): show species + image + 3 care chips immediately → "Looks right" / "Not quite"
- **No catalog match**: show a friendly fallback with options (try identifier walkthrough later, refine name, proceed anyway)
- **AI tool connected and pending**: brief "✨ Refining details..." (~3-5s timeout, then proceed) — accepts the slight delay
- **AI tool NOT connected and pending**: skip the splash entirely; land on plant detail with "connect AI" CTA already visible

This makes the splash useful for confirmation rather than a fake loader.

- [ ] **Step 3: Update AddPlant.tsx state machine**

After POST /api/plants succeeds, the response includes `catalog_slug`, `species`, `illustration_path`, `enrichment_status`. Use these to choose the splash mode:

```tsx
const aiConnected = useAiConnection().connected; // from Wave 9 Task 3 hook

if (createdPlant.catalog_slug) {
  // Catalog hit — go straight to success
  setMode('success');
  setPreview(buildPreviewFromPlant(createdPlant));
} else if (aiConnected) {
  // No catalog match, AI tool active — wait briefly for the callback
  setMode('enriching');
  pollForEnrichment(createdPlant.id, 5000); // 5s budget, then fallback
} else {
  // No catalog match, no AI tool — friendly fallback, let the user proceed
  setMode('no-match');
}
```

`pollForEnrichment` is a small helper: poll `GET /api/plants/:id` every 1s, when `enrichment_status === 'complete'` set mode='success'; on timeout go to mode='no-match'.
